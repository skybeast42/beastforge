# tracks/00_insights/

**Beast-laid, reserved** (the `00_` marker). The capture channel for noteworthy
findings — surprises, non-obvious workarounds, decisions with hidden depth,
task-closing gotchas — plus generated reports (the `/tokens` token tally lands
here as `token_usage.md`).

- One file per type (`<type>.md`); *what* to capture + the triggers are configured
  in `guidance/02_derived.md`'s *Active insight captures* section
  (template/meta-feedback is default-on). The capture *cadence* (proactive, every-K,
  at-boundaries) is a working preference in `guidance/03_user.md`'s *Working cadence*.
- Consumed by `/deck` (the harvest the deck draws from).

Capture proactively as work happens — not only at publication time.
