description: Reorganise a messy README into the canonical shape; relocate stray todos
---

Tidy `README.md`. **Do not assume it currently follows the template** — it is often a todo dump or loose notes.

1. Read the current `README.md` and `CLAUDE.md`.
2. Extract the real project identity — name + one-sentence description — for the top block.
3. Reorganise into the canonical shape (project identity → how it's organised → working in it → reproducing the work). Preserve genuine project content; improve clarity and structure.
4. **Relocate misplaced content — never delete it.** Todo-ish items are not README content; propose moving them to the right channel:
   - a next action → `STATE.md`
   - a unit of work → a `/brief` or `/ticket`
   - a settled call → `/decide`
   Confirm relocations with the User.
5. **Non-destructive**: propose the rewrite for approval *before* overwriting. Never silently drop content.
